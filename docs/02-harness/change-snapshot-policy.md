# Change Snapshot Policy

## Purpose

This policy helps future agents quickly understand project state after each change.

## Required Snapshot Updates

After every completed change, update:

```text
docs/00-project/current-state.md
```

## Snapshot Content

Record:

- implemented capabilities
- changed files
- new tests
- changed storage contracts
- known gaps
- next recommended work

## OpenSpec Changes

For OpenSpec-driven work, ensure completed changes are archived according to project policy.

## Bugfix Changes

For bugfixes, record:

- symptom
- root cause
- regression test
- affected module
- risk
